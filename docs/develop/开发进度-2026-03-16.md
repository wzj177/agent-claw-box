# 开发进度归档（2026-03-16）

## 本次目标
- 修复 OpenClaw Native 实例创建后健康检查失败问题。
- 同时支持两种终端能力：本地 Shell 与 Web Shell。
- 提升日志可读性（去除 ANSI 颜色控制符、长行折行）。

## 已完成

### 1. OpenClaw Native 启动链路修复
- 延长创建/升级超时至 30 分钟，避免长安装任务误判失败。
- Native 启动命令统一加强：
  - 启动前强制创建日志文件。
  - 固定注入 PATH（含 `~/.npm-global/bin`）。
  - 注入 `AGENTBOX_PORT`，确保进程监听端口与实例分配端口一致。
- 修复 OpenClaw 参数兼容：
  - `--bind` 从不兼容的参数调整为可用值。
  - 模板 `start_cmd` 改为读取 `${AGENTBOX_PORT:-18789}`。

### 2. 日志可读性优化
- 在后端 `get_agent_logs` 返回前统一清洗：
  - 去除 ANSI 控制符（颜色码等）。
  - 对超长行进行折行（每行 240 字符）。
- 结果：前端日志面板不再出现转义符污染，长行更易读。

### 3. 终端能力升级（双通道）
- 保留并增强“本地 Shell”：
  - Native 实例点击终端可直接打开本地终端并连接 VM。
  - 补齐 macOS / Windows / Linux 三端 provider 行为。
- 新增“Web Shell”：
  - 新页面：`/shell/:id`（xterm.js）。
  - 新增后端命令：`run_agent_shell_command`。
  - 支持命令输入与结果回显（命令执行模式）。
  - Agent 卡片新增 Web Shell 入口按钮。

### 4. 依赖与构建适配
- 新增依赖：
  - `xterm@^5.3.0`
  - `@xterm/addon-fit@^0.11.0`
- 解决版本不可用问题后，前后端相关文件错误已清理。

## 当前状态
- OpenClaw Native 实例可启动，日志可读，问题定位链路完整。
- 终端能力已支持：
  - 本地 Shell（系统终端）
  - Web Shell（内置页面）
- Web Shell 当前为“命令执行模式”，非完整 PTY 交互模式。

## 后续建议
1. 升级 Web Shell 为 PTY 会话模式（支持 `top` / `vim` 等交互程序）。
2. 增加 Web Shell 会话管理：并发限制、TTL、审计日志。
3. 为 OpenClaw 增加启动后自检步骤（`gateway probe/health`），失败时回写更明确错误。
4. 规划 Windows Ubuntu Desktop + noVNC 方案（按 VmProvider 扩展实现）。
