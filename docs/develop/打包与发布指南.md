# AgentBox 打包与发布指南

本文档用于 macOS 和 Windows 的本地打包、安装包验收和签名准备。

## 1. 前置条件

### 通用

```bash
node -v
pnpm -v
cargo --version
```

期望：
- Node.js 22.x
- pnpm 可用
- Rust stable 可用

安装依赖：

```bash
cd apps/desktop
pnpm install
```

## 2. 打包命令

在 `apps/desktop` 目录执行：

```bash
pnpm tauri:build
```

平台专用命令：

```bash
# macOS，产出 .app 和 .dmg
pnpm package:mac

# 仅构建 .app
pnpm package:mac:app

# 仅构建 .dmg
pnpm package:mac:dmg

# Windows，产出 NSIS 安装包和 MSI
pnpm package:windows

# 仅构建 NSIS
pnpm package:windows:nsis

# 仅构建 MSI
pnpm package:windows:msi
```

## 3. 产物目录

打包产物默认位于：

```bash
apps/desktop/src-tauri/target/release/bundle/
```

常见目录：
- `app/`：macOS `.app`
- `dmg/`：macOS `.dmg`
- `nsis/`：Windows NSIS 安装包 `.exe`
- `msi/`：Windows `.msi`

## 4. 当前忽略规则

Tauri 已通过根目录 `.taurignore` 排除以下内容，不参与监听和打包上下文：

- `docs/`
- 所有 `agent.md`
- 所有以 `.` 开头的目录

## 5. macOS 签名占位

当前仓库已经具备完整打包配置，但未内置证书。正式分发前建议准备以下信息：

```bash
# Apple Developer 签名身份示例
export APPLE_SIGNING_IDENTITY="Developer ID Application: Your Company (TEAMID)"

# 如果后续接入 notarization，可补充以下占位
export APPLE_ID="your-apple-id@example.com"
export APPLE_TEAM_ID="TEAMID"
export APPLE_APP_SPECIFIC_PASSWORD="app-specific-password"
```

建议：
- 开发测试包可以先不签名，确认功能链路。
- 对外分发的 `.dmg` 再接入 Developer ID 签名和 notarization。

## 6. Windows 签名占位

如果后续要减少 SmartScreen 告警，建议准备代码签名证书。当前阶段可先产出未签名安装包进行功能测试。

可预留的信息：

```powershell
$env:WINDOWS_CERT_FILE="C:\certs\agentbox.pfx"
$env:WINDOWS_CERT_PASSWORD="your-password"
```

建议：
- 内部测试阶段先验证安装、启动、升级、卸载。
- 外部分发再接入 `signtool` 或 CI 签名流程。

## 7. macOS 验收清单

1. 执行 `pnpm package:mac` 成功。
2. 确认 `bundle/app` 下生成 `AgentBox.app`。
3. 确认 `bundle/dmg` 下生成 `.dmg`。
4. 双击 `.app` 能启动主界面。
5. 首次启动可以完成运行环境检查与初始化。
6. OpenClaw 实例可以打开、保存配置、打开控制台。
7. 监控页能看到 CPU、内存、网络数据。
8. 退出并重启应用后，实例状态能正确恢复。

## 8. Windows 验收清单

1. 在 Windows 机器执行 `pnpm package:windows` 成功。
2. 确认 `bundle/nsis` 生成安装包 `.exe`。
3. 确认 `bundle/msi` 生成 `.msi`。
4. NSIS 安装器可以正常安装到默认目录。
5. 应用启动后主界面正常显示，无白屏。
6. 创建 OpenClaw 实例后可进入运行中状态。
7. 配置页切换 provider 后，模型字段会自动同步。
8. 打开 dashboard 不出现 token 错误或乱码 URL。
9. 监控页持续刷新且重启后状态不残留异常。
10. 卸载后主程序目录被清理，重新安装可正常启动。

## 9. 建议的发布顺序

1. 先在 macOS 执行 `pnpm package:mac`，验证本地产物与启动链路。
2. 再到 Windows 环境执行 `pnpm package:windows`，验证安装、运行、卸载。
3. 验证无阻塞问题后，再补签名和自动化发布。
